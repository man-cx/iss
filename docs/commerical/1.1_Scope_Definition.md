## 5. Implementation Phases

The project follows SAP Activate methodology with the following Q-Gate phases:

### 5.1 Prepare Phase
**Focus:** Project initiation & planning

**ACLOUDEAR Tasks:**
- Conduct initial meeting between Customer's Project Manager and ACLOUDEAR's Project Manager to review project scope and preparation activities
- Conduct project team kickoff (project objectives, roles & responsibilities, project schedule)
- Confirm solution design of S4HC business scenarios utilizing workshop(s)
- Confirm organizational management structure design utilizing workshop(s)
- Provide "Key Users Enablement Roadmap" accelerator
- Review overall data migration approach, data migration tool and related documentation
- Introduce Customer to ACLOUDEAR provided data migration templates
- Align data migration project milestones with overall project schedule

**ISS Responsibilities:**
- Ensure key users complete all relevant S4HC learning tasks required for Prepare phase
- Provide information for organization structure (Organizational Management) setup
- Perform data cleansing as required in source (legacy) system(s)
- Extract data from source (legacy) system(s)
- Complete Scope Verification

### 5.2 Explore Phase
**Focus:** Solution configuration

**ACLOUDEAR Tasks:**
- Setup organization structure in S4HC Solution
- Perform Explore activities in S4HC Go-Live Activity List
- Review with ISS and Customer any activities requiring Customer input
- Perform Solution Acceptance Assessment

**ISS Responsibilities:**
- Ensure key users have completed all relevant S4HC learning tasks
- Ensure Key Users plan and perform End User enablement
- Populate data migration templates with cleansed legacy data
- Perform upload of data migration templates via S4HC Data Migration Tool with ACLOUDEAR support
- Perform S4HC manual data entry, if applicable
- Verify and reconcile migrated data
- Provide Solution Acceptance Confirmation

### 5.3 Realize Phase
**Focus:** Testing & validation

**ACLOUDEAR Tasks:**
- Provide test plan and standard test scenarios based on business scope
- Assist with preparation and adaptation of test plan and test scenarios utilizing workshop(s)
- Assist and support ISS and Customer with resolution of issues related to test plan completion
- Assist Customer with planning cutover activities utilizing workshop(s)
- Define cutover schedule jointly with ISS and Customer
- Perform Go-Live Readiness Assessment

**ISS Responsibilities:**
- Ensure Key Users have completed all relevant S4HC learning tasks required for Test phase
- Modify test plan to include any required Customer specific scenarios
- Assign test activities to responsible Key Users
- Perform integration testing and document test results in SAP provided S4HC Test Plan
- Finalize cutover schedule and allocate resources to cutover activities
- Provide Go-Live Readiness Acceptance confirmation

### 5.4 Go-Live Phase
**Focus:** Production cutover

**ACLOUDEAR Tasks:**
- Monitor and support Customer cutover activities as per S4HC Cutover Schedule
- Prepare and issue Acceptance Protocol for Customer sign-off
- Set S4HC Production System status to "Live"

**ISS Responsibilities:**
- Ensure Key Users and End Users have completed all relevant S4HC Learnings
- Perform organizational change management activities
- Execute cutover activities as per S4HC Cutover Schedule:
  - Populate data migration templates with cleansed legacy data
  - Perform upload of data migration templates via S4HC Data Migration Tool, if applicable
  - Perform S4HC manual data entry, if applicable
  - Verify and reconcile migrated data
- Execute final activities in S4HC Go-Live Activity List
- Review, sign and return Acceptance Protocol to ACLOUDEAR

## 10. Acceptance Criteria

### 10.1 Acceptance Process

- ISS, on receipt of notification from ACLOUDEAR that Deliverable is ready for acceptance, shall provide written notification within reasonable Business Days
- ISS shall notify ACLOUDEAR that Deliverable has either been Accepted or Rejected
- If rejected, ACLOUDEAR shall attempt to rectify defects within period commensurate with severity
- Customer shall re-test Deliverable within ten (10) Business Days after defect rectification

### 10.2 Deemed Acceptance

- If ISS does not serve notice of acceptance within specified period and does not indicate rejection, Customer will be deemed to have accepted the Deliverable
- A Deliverable shall also be deemed accepted if ISS uses the Deliverable in live operation for a period in excess of ten (10) days

### 10.3 Acceptance Protocol

- ACLOUDEAR shall prepare and issue Acceptance Protocol for Customer sign-off
- ISS shall review, sign and return Acceptance Protocol to ACLOUDEAR
